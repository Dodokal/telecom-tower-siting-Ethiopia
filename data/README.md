# Data directory

## What's tracked in Git

- `predictor_metadata.csv` — machine-readable list of all 23 predictor variables, sources, licences, URLs
- `hyperparameters.yaml` — tuned hyperparameter values for each base learner, search ranges, CV configuration, priority thresholds
- `seeds.yaml` — every random seed used in the pipeline
- `sessionInfo.txt` — R version and package versions for the analysis run (generated by `R/00_setup.R`)

## What's NOT tracked (and how to regenerate it)

The actual raster predictor data and tower presence/absence files are too large for Git and in several cases are under licences that prevent redistribution. Re-create them as follows:

### Earth observation rasters (~1.5 GB total)

Most layers are exported from Google Earth Engine. The R/01_assemble_presence.R and R/02_build_training_table.R scripts include the Earth Engine asset IDs and download paths. You will need a Google Earth Engine account (free for non-commercial use):

1. Authenticate Earth Engine: `rgee::ee_Initialize()`
2. Run the export commands at the top of `01_assemble_presence.R`
3. Place the exported GeoTIFFs in `ETH_towers/` (mirror the structure used by the pipeline)

Alternatively, download each layer directly from the URL in `predictor_metadata.csv`.

### Tower presence points

Re-generate from the Geofabrik Ethiopia OSM extract:

```bash
wget https://download.geofabrik.de/africa/ethiopia-latest.osm.pbf -P data/
```

Then in R:
```r
source("R/01_assemble_presence.R")
```

This extracts `man_made=tower` (with `tower:type=communication`) and `man_made=mast` features, applies 75 m spatial deduplication, and saves the final 357 presences to `data/presence_absence.gpkg`.

### DHS 2016 Ethiopia data (registered access)

DHS data cannot be redistributed but can be downloaded for free after registration:

1. Register at https://dhsprogram.com (free)
2. Request the Ethiopia 2016 DHS Standard survey
3. Download the IR (Individual Recode) and HR (Household Recode) datasets and the GE (Geographic) cluster shapefile
4. Place all files in `data/dhs_2016/` (this directory is excluded from Git)
5. `R/02_build_training_table.R` will build the cluster-level internet-use and electricity-access surfaces from these inputs

## Predictor naming convention

Continuous predictors enter the model as numeric features. Categorical predictors (LC, SMOD, PA, FLD) are one-hot encoded via the recipes-package `step_dummy()`. The resulting class indicators appear in figures with the suffix pattern shown below:

| Original | One-hot child | Meaning |
|---|---|---|
| LC | LC_CROP, LC_BLT, LC_GRASS, LC_SHRUB, LC_BARE, LC_WET | ESA WorldCover classes |
| SMOD | SMOD_RURAL, SMOD_UNINHAB, SMOD_PERI, SMOD_DEN, SMOD_URB | GHSL settlement classes |
| PA | (binary, no suffix) | Protected area presence |
| FLD | (binary, no suffix) | Flood-prone pixel |
